OpenClaw 沙箱原理

只隔离 Agent 的工具执行——LLM 对话、路由调度等核心逻辑始终在宿主机运行。一个可选的安全增强层，而非运行时依赖。

OpenClaw 的沙箱仅隔离 exec、read、write 等工具执行，Gateway 进程本身、LLM 对话、路由调度等核心逻辑始终在宿主机运行。只隔离了 Agent 的工具执行。

01沙箱模式

沙箱通过 sandbox.mode 配置控制，有三种模式可选：

模式行为
off（默认）不使用沙箱，所有工具直接在宿主机执行
non-main仅对非主会话沙箱（群组/频道会话被沙箱，主会话不沙箱）
all所有会话都在沙箱中执行
💡 提示：默认关闭时所有工具就像普通程序一样直接在宿主机执行，完全不需要 Docker。只有主动配置 sandbox.mode = "all" 或 "non-main" 时才需要 Docker。

02沙箱里执行哪些工具？

沙箱内可以执行 shell 命令、读写文件、管理子会话，但不能使用浏览器、定时任务、网关管理、消息通道等敏感操作。允许和拒绝的工具列表定义在 sandbox/constants.ts 中：

typescriptsandbox/constants.ts
// 默认允许的工具（13 个）
exec, process, read, write, edit, apply_patch,
image, sessions_list, sessions_history, sessions_send,
sessions_spawn, sessions_yield, subagents, session_status

// 默认拒绝的工具
browser, canvas, nodes, cron, gateway,
以及所有 channel 工具

04沙箱文件系统访问

FsBridge 是沙箱文件操作的安全代理，负责在容器内安全地执行读写删改，同时防范路径逃逸和 TOCTOU 攻击。OpenClaw 自己实现，非第三方工具。

⚠️ 注意：OpenClaw 没有实现同步机制，在 none 网络模型下，操作的文件都需要手动同步。

05沙箱安全边界

5.1 被禁止的 bind 挂载路径

以下路径绝不允许挂载进沙箱容器，包括 Docker socket（挂载等于交出宿主机控制权）和用户凭据目录：

typescriptvalidate-sandbox-security.ts
const BLOCKED_HOST_PATHS = [
  "/etc", "/private/etc", "/proc", "/sys", "/dev",
  "/root", "/boot", "/run", "/var/run",
  "/private/var/run", "/var/run/docker.sock",
  "/private/var/run/docker.sock", "/run/docker.sock",
];

const BLOCKED_HOME_SUBPATHS = [
  ".aws", ".cargo", ".config", ".docker",
  ".gnupg", ".netrc", ".npm", ".ssh",
];

5.2 被禁止的网络模式

网络模式策略
network: "host"始终被阻止
network: "container:<id>"默认阻止，需 dangerouslyAllowContainerNamespaceJoin: true

5.3 被过滤的环境变量

所有 API Key、Token、密码、私钥相关的环境变量被阻止传入沙箱。只允许基础的系统环境变量（LANG、PATH、HOME 等）通过：

typescriptsanitize-env-vars.ts
const BLOCKED_ENV_VAR_PATTERNS: ReadonlyArray<RegExp> = [
  // 各平台 API Key
  /^ANTHROPIC_API_KEY$/i,
  /^OPENAI_API_KEY$/i,
  /^GEMINI_API_KEY$/i,
  /^OPENROUTER_API_KEY$/i,
  /^MINIMAX_API_KEY$/i,
  /^ELEVENLABS_API_KEY$/i,
  // 通信 Token
  /^TELEGRAM_BOT_TOKEN$/i,
  /^DISCORD_BOT_TOKEN$/i,
  /^SLACK_(BOT|APP)_TOKEN$/i,
  /^LINE_CHANNEL_(SECRET|ACCESS_TOKEN)$/i,
  /^OPENCLAW_GATEWAY_(TOKEN|PASSWORD)$/i,
  // 云凭证
  /^AWS_(SECRET_ACCESS_KEY|SECRET_KEY|SESSION_TOKEN)$/i,
  /^(GH|GITHUB)_TOKEN$/i,
  /^(AZURE|AZURE_OPENAI|COHERE|AI_GATEWAY|OPENROUTER)_API_KEY$/i,
  // 通用敏感模式
  /_?(API_KEY|TOKEN|PASSWORD|PRIVATE_KEY|SECRET)$/i,
];

const ALLOWED_ENV_VAR_PATTERNS: ReadonlyArray<RegExp> = [
  /^LANG$/, /^LC_.*$/i, /^PATH$/i,
  /^HOME$/i, /^USER$/i, /^SHELL$/i,
  /^TERM$/i, /^TZ$/i, /^NODE_ENV$/i,
];
